# Contributing to Zeroth Studios Code Exercises

Thanks for your interest in contributing! These exercises are companion material for [Zeroth Studios](https://zerothstudios.com) podcasts.

## How to Contribute

### Report a Bug

If an exercise has a typo, produces an error, or gives confusing output, [open an issue](https://github.com/zerothstudios/zeroth-code/issues) with:

- Which exercise file (e.g., `zeroth-ai/season-1/ep01-what-makes-something-smart/exercises.py`)
- What you expected to happen
- What actually happened
- Your Python version (`python3 --version`)

### Suggest an Improvement

Have an idea for a better explanation or an additional exercise? Open an issue describing what you'd like to see and why it would help learners.

### Submit a Pull Request

1. Fork this repository.
2. Create a branch for your change.
3. Make your changes. Keep exercises beginner-friendly — no jargon, no complex dependencies.
4. Test your changes by running the exercise file: `python3 <path-to-exercise>.py`
5. Open a pull request describing what you changed and why.

## Guidelines

- **Keep it simple.** These exercises are for people with zero programming experience. If a concept needs a paragraph of explanation before the code makes sense, it's too complex.
- **Python standard library only.** No external packages. Exercises should run with just Python installed — no `pip install` required.
- **One file per episode.** Each episode gets one `exercises.py` file that covers the key concepts from that episode.
- **Match the episode.** Exercises should reinforce what the podcast episode teaches. Listen to the episode before writing exercises for it.

## Code Style

- Use clear variable names over short ones.
- Include print statements that explain what the code is doing and why.
- Use comments sparingly — the print output should do most of the teaching.
